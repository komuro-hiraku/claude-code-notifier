# claude-code-notifier 要件定義書（軽量版）

## 概要

macOSメニューバー常駐アプリで、複数のClaude Codeタスクを監視し、回答待ち・指示待ち・タスク完了の状態を通知するソリューション。

**目的**: Claude Codeの複数タスク実行時に、回答待ちや指示待ちで停止していることに気づけない問題を解決する。

## 簡易ユーザーストーリー

### ストーリー1: 回答待ち状態の通知

**私は** Claude Codeを複数並行使用する開発者 **として**
**回答待ち状態になったタスクを即座に知りたい**
**そうすることで** 作業のブロッキングを防ぎ、効率的に複数タスクを管理できる

**関連要件**: REQ-002, REQ-101

### ストーリー2: 指示待ち状態の通知

**私は** Claude Codeで複数タスクを実行する開発者 **として**
**タスクが完了し次の指示を待っていることを知りたい**
**そうすることで** 次の作業指示をすぐに出せる

**関連要件**: REQ-003, REQ-102

### ストーリー3: タスク完了の通知

**私は** Claude Codeで長時間実行タスクを起動する開発者 **として**
**タスク完了を即座に知りたい**
**そうすることで** 他の作業をしながらでも、完了のタイミングを見逃さない

**関連要件**: REQ-004, REQ-103

## 基本的な受け入れ基準

### REQ-001: ログファイル監視

**Given（前提条件）**: Claude Codeが実行中で、ログファイルが存在する
**When（実行条件）**: ログファイルに新しいエントリが追加される
**Then（期待結果）**: システムは変更を検出し、内容を解析する

**テストケース**:
- [ ] 正常系: ログファイルに新規エントリが追加された際に検出される
- [ ] 主要な異常系: ログファイルが存在しない場合、エラーログを出力する

### REQ-002: 回答待ち状態の検出

**Given（前提条件）**: ログファイルを監視中
**When（実行条件）**: 回答待ち状態を示すログエントリを検出
**Then（期待結果）**: 該当タスクの情報を抽出する

**テストケース**:
- [ ] 正常系: 回答待ちパターンのログを正しく識別する
- [ ] 主要な異常系: 不明なログフォーマットの場合、スキップする

### REQ-005: macOS通知センター通知

**Given（前提条件）**: 検出すべき状態（回答待ち/指示待ち/タスク完了）が発生
**When（実行条件）**: 通知送信処理が実行される
**Then（期待結果）**: macOS通知センターに通知が表示される

**テストケース**:
- [ ] 正常系: 通知が正しいタイトル・メッセージで表示される
- [ ] 主要な異常系: 通知権限がない場合、ユーザーに権限要求を表示する

### REQ-006: メニューバー常駐

**Given（前提条件）**: アプリケーションが起動
**When（実行条件）**: macOSのメニューバーにアイコンを配置
**Then（期待結果）**: アイコンがメニューバーに表示され、クリックでメニューが開く

**テストケース**:
- [ ] 正常系: メニューバーにアイコンが表示される
- [ ] 正常系: アイコンクリックで監視状態や設定メニューが表示される

## 最小限の非機能要件

- **パフォーマンス**: ログファイル監視はCPU使用率1%未満であるべき 🟡 *軽量動作の一般的な目標値*
- **パフォーマンス**: 通知は状態変化から1秒以内に表示されるべき 🟡 *ユーザ体感として妥当な推測*
- **セキュリティ**: ログファイルへのアクセス権限をユーザーに明示すべき 🟡 *一般的なセキュリティ慣行*
- **ユーザビリティ**: メニューバーアイコンは直感的で小さく表示されるべき 🟡 *macOSデザインガイドラインから推測*
- **ユーザビリティ**: 通知は邪魔にならず、かつ見落とされないバランスが必要 🟡 *UX原則から推測*
