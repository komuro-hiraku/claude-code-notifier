# claude-code-notifier 準備タスク（ユーザー作業）

> **仕様**: [requirements.md](requirements.md)
> **生成日**: 2026-03-26

**【信頼性レベル凡例】**:
- 🔵 **青信号**: 要件定義書・設計文書・ユーザヒアリングで明確に必要と判明したタスク
- 🟡 **黄信号**: 要件定義書・設計文書から妥当に推測されるタスク
- 🔴 **赤信号**: 推測による予防的タスク（実装時に不要と判明する可能性あり）

## 必須（実装開始前に完了が必要）

以下のタスクが完了していないと、実装フェーズでブロッカーになります。

- [ ] **Claude Codeのログファイル位置の特定** 🔵 *ヒアリング Q5より*
  - Claude Codeが出力するログファイルの場所を特定してください
  - 調査方法の例:
    - Claude Codeのドキュメントを確認
    - `find ~ -name "*.log" -type f 2>/dev/null | grep -i claude` でログファイルを検索
    - Claude Codeの設定ファイルから確認
    - 一般的な候補: `~/.claude/logs/`, `~/Library/Logs/Claude/`, `~/.config/claude/logs/`
  - 関連要件: REQ-001

- [ ] **ログファイルのフォーマット調査** 🔵 *REQ-001から必須*
  - ログファイルの形式（JSON、プレーンテキスト、構造化ログ等）を確認
  - ログエントリのタイムスタンプフォーマットを確認
  - ログローテーションの有無と方式を確認
  - 調査方法:
    - 実際のログファイルを開いて内容を確認
    - サンプルログエントリを数件記録
  - 関連要件: REQ-001

- [ ] **状態検出パターンの特定** 🔵 *REQ-002, REQ-003, REQ-004から必須*
  - 以下の各状態を示すログエントリのパターン（キーワード、文字列）を特定:
    - 回答待ち状態
    - 指示待ち状態
    - タスク完了
  - 調査方法:
    - Claude Codeを実際に操作しながらログを確認
    - 各状態になった際のログエントリを記録
    - 正規表現パターンを作成
  - 関連要件: REQ-002, REQ-003, REQ-004

## 推奨（実装中に用意できればOK）

実装を開始できますが、該当機能の実装前までに準備してください。

- [ ] **Xcode開発環境のセットアップ** 🟡 *REQ-401 Swift実装から推測*
  - Xcodeの最新版をインストール
  - Swift開発環境が動作することを確認
  - 必要になるフェーズ: Phase 1（プロジェクトセットアップ）
  - 関連要件: REQ-401

- [ ] **macOS通知センターAPIのドキュメント確認** 🟡 *REQ-005から推測*
  - UserNotifications frameworkのドキュメントを確認
  - 通知権限の要求方法を確認
  - サンプルコードを確認
  - 参考URL: https://developer.apple.com/documentation/usernotifications
  - 必要になるフェーズ: Phase 2（通知機能実装）
  - 関連要件: REQ-005

- [ ] **ファイル監視APIのドキュメント確認** 🟡 *REQ-001から推測*
  - FSEvents APIまたはDispatchSourceのドキュメントを確認
  - ファイル変更監視の実装方法を確認
  - 参考URL: https://developer.apple.com/documentation/coreservices/file_system_events
  - 必要になるフェーズ: Phase 1（ログ監視機能実装）
  - 関連要件: REQ-001

- [ ] **メニューバーアプリのドキュメント確認** 🟡 *REQ-006から推測*
  - NSStatusBar, NSStatusItemのドキュメントを確認
  - メニューバーアプリのサンプルコードを確認
  - 参考URL: https://developer.apple.com/documentation/appkit/nsstatusbar
  - 必要になるフェーズ: Phase 1（メニューバー常駐実装）
  - 関連要件: REQ-006

## 確認事項（判断が必要）

実装方針に影響するため、早めの判断・確認が推奨されます。

- [ ] **複数Claude Codeインスタンスの監視** 🟡 *REQ-001から推測*
  - Claude Codeを複数同時実行した場合、ログファイルはどうなるか？
    - 選択肢1: 同一ログファイルに複数タスクの情報が混在
    - 選択肢2: タスクごとに別々のログファイルが生成される
    - 選択肢3: 最新のインスタンスのみログ出力
  - この判断により、監視対象ファイル数やタスク識別方法が変わる
  - 関連要件: REQ-001, REQ-002, REQ-003, REQ-004

- [ ] **通知の重複制御** 🟡 *REQ-101, REQ-102, REQ-103から推測*
  - 同一状態が連続して検出された場合、通知を毎回表示するか？
    - 選択肢1: 毎回通知（うるさい可能性）
    - 選択肢2: 一定時間内は重複通知しない
    - 選択肢3: 状態が変化した場合のみ通知
  - ユーザー体験に影響するため、早めの判断が必要
  - 関連要件: REQ-101, REQ-102, REQ-103
